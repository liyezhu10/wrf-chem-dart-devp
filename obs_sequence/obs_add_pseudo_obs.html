Overview

This specialized tool is intended to be used with the Quadratic Squared Error filter option. See the filter documentation for more details on the usage.

Running this tool will add an additional observation of type QUAD_FILTER_SQUARED_ERROR after every observation in the input observation sequence file. It is recommended that running this tool be the last step before running filter, since using some of the other observation sequence tools may separate the Pseudo Observations from their corresponding pair.

The standard obs_sequence_tool can be used to remove the Pseudo Observations after running filter and before running any observation space diagnostics. A suggested namelist is: 

&obs_sequence_tool_nml
   filename_seq    = 'obs_seq_with_pseudo_obs.final'
   filename_out    = 'obs_seq.final',
   keep_types      = .false.,
   obs_types       = 'QUAD_FILTER_SQUARED_ERROR'
   gregorian_cal   = .true.
/

NAMELIST

This namelist is read from the file input.nml. Namelists start with an ampersand '&' and terminate with a slash '/'. Character strings that contain a '/' must be enclosed in quotes to prevent them from prematurely terminating the namelist. 

&obs_add_pseudo_obs_nml
   filename_in  = 'obs_seq.out',
   filename_out = 'obs_seq_with_pseudo_obs.out',
   print_only   = .false.,
   calendar     = 'Gregorian'
/


Item Type Description
filename_in character(len=256) The name of an observation sequence file.
filename_out character(len=256) The name of the output observation sequence file to be created. If it exists it will be overwritten. It will contain all the observations in the input file along with a corresponding Pseudo Observation for each existing observation, doubling the size of the observation sequence file. The specific type of each new observation will be QUAD_FILTER_SQUARED_OBS.
print_only logical If .true., print out the counts and types of each of the output observations, without creating the output file.
calendar character(len=32) Name of the calendar to use. Only controls how the log message print statements are formatted; if 'Gregorian' the time is printed as a calendar year, month, date, etc. If 'NO CALENDAR', the time is printed as days and seconds only.

BUILDING

Some $DART/models/*/work directories will build the tool along with other executable programs. The &preprocess_nml namelist must include "../../../obs_def/obs_def_quad_filter_mod.f90" along with any other forward operator/observation definition files. If mkmf_obs_add_pseudo_obs or path_names_obs_add_pseudo_obs are not found in the work directory, copy them from the obs_sequence directory and adapt the path_names file to match the rest of the files for that model. The preprocess program must be built and run first, to define the supported observation types. See the preprocess documentation for more details on how to define the list and run it. The combined list of all observation types which will be encountered over all input files must be in the preprocess input list. The other important choice when building the tool is to include a compatible locations module. For the low-order models, the oned module should be used; for real-world observations, the threed_sphere module should be used.

Generally the directories where executables are built will include a "quickbuild.csh" script which will build and run preprocess and then build the rest of the executables. The "input.nml" namelists will need to be edited to include all the required observation types first.
